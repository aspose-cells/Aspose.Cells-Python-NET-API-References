##  RenameStrategy перечисление
Вариант стратегии для повторяющихся названий столбцов.



Тип RenameStrategy предоставляет следующие элементы:

###  Поля
| Поле| Описание|
| :- | :- |
| EXCEPTION | Выбрасывает исключение.|
| DIGIT | Назван с цифрой. Повторяющиеся имена станут ...1, ...2 и т. д.|
| LETTER | Назван с буквой. Повторяющиеся имена станут ...A, ...B и т. д.|



###  Примечания

При обработке данных с заголовками в некоторых сценариях требуется, чтобы заголовки не дублировались для всех столбцов.
Например, при экспорте данных в таблицу данных и заголовок должен использоваться в качестве имени столбца таблицы данных,
повторяющиеся значения заголовка недействительны.
В таких ситуациях пользователь может определить, как с ними справиться, указав эту стратегию.
