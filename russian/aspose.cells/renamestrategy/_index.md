##  RenameStrategy перечисление
Опция стратегии для повторяющихся имен столбцов.



Тип RenameStrategy предоставляет следующие члены:

###  Поля
| Поле| Описание|
| :- | :- |
| EXCEPTION | Выдает исключение.|
| DIGIT | Именуется цифрой. Повторяющиеся имена станут ...1, ...2 и т. д.|
| LETTER | Именуется буквой .. Повторяющиеся имена станут ... A, ... B и т. д.|



###  Примечания

При обработке данных с заголовками в некоторых сценариях требуется, чтобы заголовки не дублировались для всех столбцов.
Например, при экспорте данных в таблицу данных и заголовок должен быть взят как имя столбца таблицы данных,
повторяющиеся значения заголовка недействительны.
Для таких ситуаций пользователь может определить, как с ними справиться, указав эту стратегию.
