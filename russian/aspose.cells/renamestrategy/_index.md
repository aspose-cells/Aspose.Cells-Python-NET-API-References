##  RenameStrategy перечисление
Вариант стратегии для повторяющихся имен столбцов.



Тип RenameStrategy предоставляет следующие элементы:

###  Поля
| Поле| Описание|
| :- | :- |
| EXCEPTION | Выдает исключение.|
| DIGIT | Имя содержит цифру. Повторяющиеся имена будут выглядеть так: ...1, ...2 и т.д.|
| LETTER | Названия содержат буквы. Дублирующиеся названия будут выглядеть так: ...A, ...B и т. д.|



###  Примечания

При обработке данных с заголовками в некоторых сценариях требуется, чтобы заголовки не дублировались для всех столбцов.
Например, при экспорте данных в таблицу данных заголовок должен быть указан как имя столбца таблицы данных,
Дублирующиеся значения заголовка недействительны.
Для подобных ситуаций пользователь может определить, как с ними справиться, указав данную стратегию.
