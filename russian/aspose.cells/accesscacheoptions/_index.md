##  AccessCacheOptions перечисление
Параметры кэша для доступа к данным. Можно комбинировать с | оператор для нескольких вариантов вместе.



Тип AccessCacheOptions предоставляет следующие элементы:

###  Поля
| Поле| Описание|
| :- | :- |
| NONE | Нет кеша для доступа к данным.|
| ALL | Примените все возможные оптимизации для всех видов доступа к данным в книге.<br/> Все настройки и данные не должны изменяться во время оптимизированного доступа.|
| POSITION_AND_SIZE |Примените возможную оптимизацию для получения положения и размера объекта (например, формы).<br/> Настройки высоты строки и ширины столбца не следует изменять во время оптимизированного доступа.|
| CELLS_DATA | Примените возможную оптимизацию для получения значений ячеек.<br/>Данные Cells (данные и настройки Cell, строка) не следует изменять во время<br/>оптимизированный доступ, новые объекты Cell/Row также не должны создаваться (например,<br/> с помощью [индексатора]).|
| CELL_DISPLAY | Примените возможную оптимизацию для получения результатов, связанных с отображением<br/>ячейки ([`Cell.display_string_value`](/cells/python-net/ru/aspose.cells/cell#display_string_value), [`Cell.get_style`](/cells/python-net/ru/aspose.cells/cell/get_style), [`Cell.get_display_style`](/cells/python-net/ru/aspose.cells/cell/get_display_style) и т. д.).<br/>Cells Объекты, связанные с данными и стилями (Cell/стили строк/столбцов, ширина столбца и т. д.), не следует изменять.<br/> во время оптимизированного доступа.|
| GET_FORMULA | Примените возможную оптимизацию для получения формул.<br/>Все данные и настройки, которые могут повлиять на выражение формулы (имя рабочего листа, текст имени,<br/> столбец таблицы и т. д.) не следует изменять во время оптимизированного доступа.|
| SET_FORMULA |Примените возможную оптимизацию для настройки формул.<br/>Все данные и настройки, которые могут повлиять на выражение формулы (имя рабочего листа, текст имени,<br/> столбец таблицы и т. д.) не следует изменять во время оптимизированного доступа.|
| CALCULATE_FORMULA | Примените возможную оптимизацию для расчета формул.<br/>Cells данные не должны изменяться при оптимизированном доступе, ни один новый объект (Cell, строка и т. д.)<br/> также должен быть создан (например, с помощью [indexer]).|
| CONDITIONAL_FORMATTING | Примените возможную оптимизацию для получения результата условного форматирования.<br/>Все данные и настройки, которые могут повлиять на результат условного форматирования (настройки<br/> условное форматирование, зависимые значения ячеек и т. д.) не следует изменять во время оптимизированного доступа.|
| VALIDATION | Примените возможную оптимизацию для получения результата проверки.<br/>Все данные и настройки, которые могут повлиять на результат валидации(настройки валидации,<br/> значения зависимых ячеек и т. д.) не следует изменять во время оптимизированного доступа.|



###  Примечания

Для некоторых функций доступ к большому набору данных требует множества повторяющихся и сложных операций.
такие как поиск, расчет и т. д., и эти операции займут много дополнительного времени.
В распространенных ситуациях все зависимые данные остаются неизменными во время доступа, поэтому можно создать некоторые кэши и использовать их для
улучшить производительность доступа.
Для этой цели мы предоставляем этот номер API, чтобы пользователь мог указать, какой тип доступа к данным необходим.
быть оптимизирован с помощью возможного механизма кэширования.


Обратите внимание: для разных вариантов может потребоваться, чтобы разные наборы данных были «только для чтения».
А производительность доступа к данным зависит от многих аспектов, использования механизма кэширования.
не гарантирует, что производительность будет улучшена. Для некоторых ситуаций
например, набор данных, к которому осуществляется доступ, мал, использование кэша может привести к еще большему увеличению времени, поскольку
само кэширование также требует определенного дополнительного времени.
